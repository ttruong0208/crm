2. Quy trình vận hành (khuyến nghị)

  1. Thiết lập: Admin đăng nhập → tạo tài khoản Zalo trong CRM Full → tạo mã sync từng account.
  2. Extension: Chrome → Load unpacked → tools/zalo-sync-extension → dán mã sync → mở chat.zalo.me.
  3. Import nhóm: Extension quét nhóm → CRM «Hiện nhóm vừa quét» → Import (lọc Nhóm / Cá nhân).
  4. Chiến dịch: Tạo chiến dịch → soạn tin riêng từng nhóm → gán nhân viên.
  5. Gửi tin: Nhân viên mở Zalo Web → bấm «💬 Mở Zalo» từ CRM → gửi tay → extension tự đánh dấu «Đã gửi».
  6. Theo dõi: Inbox tập trung, nhắc chăm sóc, analytics nhân viên, export CSV/JSON.

3. Danh sách chức năng đầy đủ

3.1 Đăng nhập & bảo mật

  • JWT access token + refresh token tự động
  • Dữ liệu lưu PostgreSQL theo user
  • Sync token extension: chỉ POST — không GET/DELETE dữ liệu CRM

3.2 Nhóm & Import

  • Thêm / xóa nhóm, phân loại Nhóm / Cá nhân
  • Import: Extension quét API Zalo, CSV, dán text, bookmarklet
  • Re-import cập nhật chatType, Zalo ID (updateExisting)
  • Gộp trùng SĐT giữa các tài khoản Zalo (modal khi import)
  • Hồ sơ KH: SĐT, ghi chú, thẻ, phân loại, lịch sử tương tác

3.3 Chiến dịch & Công việc

  • Tin nhắn riêng từng nhóm trong chiến dịch
  • Trạng thái: Chưa gửi → Đã gửi → Đang trả lời → Hoàn tất
  • Assignee, độ nóng, điểm lead, follow-up (+24h/+48h)
  • Lọc nâng cao + phân trang (10/25/50/100)
  • Gán hàng loạt, đánh dấu Đã gửi/Hoàn tất theo bộ lọc
  • Chèn mẫu tin / trả lời nhanh vào ô đang chọn
  • Export CSV chiến dịch
  • 💬 Mở Zalo — link trực tiếp tới chat

3.4 Broadcast (thông báo hàng loạt)

  • Một nội dung → nhiều nhóm, theo dõi đã gửi từng nhóm
  • Gửi thủ công trên Zalo Web, CRM quản lý tiến độ

3.5 CRM Full

  • Đa tài khoản Zalo + mã sync riêng từng account
  • Thẻ & phân loại KH (Lead, VIP, Đã mua…)
  • Thư viện mẫu tin & trả lời nhanh
  • Nhắc chăm sóc (inactive + follow-up quá hạn)
  • Inbox tập trung — sắp theo tương tác gần nhất
  • Webhook + Export JSON (GET /api/export/crm)

4. Tính năng vận hành (PO — đã triển khai)

4.1 Health Check Extension (Heartbeat)

  • Extension gửi POST /api/sync/heartbeat mỗi 5 phút
  • CRM hiển thị Online / Offline (ngưỡng 10 phút không có tín hiệu)
  • Vị trí: CRM Full → Tài khoản Zalo; mục Đồng bộ Zalo → danh sách heartbeat
  • Admin thấy cảnh báo khi extension offline

4.2 Gộp trùng SĐT (Deduplication)

  • Khi import, nếu SĐT đã tồn tại ở tài khoản khác → modal hỏi gộp hồ sơ
  • API: POST /api/groups/check-duplicates, POST /api/crm/merge-groups
  • Gộp: tag, lịch sử, task, broadcast; lưu linkedZaloChats[]
⚠ Gộp SĐT cần nhóm đã có số điện thoại trong hồ sơ CRM. Quét Zalo thường chưa có SĐT — nhập tay trong Hồ sơ.

4.3 Bảo mật Sync Token

Token extension chỉ được POST vào 4 endpoint:

  • /api/sync/heartbeat
  • /api/sync/scan-groups
  • /api/sync/zalo-sent
  • /api/sync/interaction

Không thể dùng token để GET danh sách khách, export, hay xóa DB qua JWT endpoints.

7. Giới hạn & lưu ý

  • Không auto gửi Zalo — cố ý, tránh rủi ro khóa tài khoản
  • Inbox CRM = dữ liệu ghi nhận qua CRM/extension, không phải inbox Zalo live đầy đủ
  • Mỗi tài khoản Zalo vẫn mở browser/profile riêng
  • Zalo Web cập nhật thường xuyên — cần theo dõi extension Online/Offline
  • Chưa deploy production — cần hosting + backup DB cho khách hàng
